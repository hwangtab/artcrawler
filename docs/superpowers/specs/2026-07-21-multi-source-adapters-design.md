# 다중 소스 어댑터 설계 — 예술지원사업 캘린더

- 날짜: 2026-07-21
- 상태: 승인됨
- 범위: 1차 (어댑터 구조 도입 + KOCCA 소스 추가)

## 배경

현재 [auto.js](../../../auto.js)는 아트누리(artnuri.or.kr) 한 곳만 수집한다. 아트누리는
자체가 101개 기관(ARKO, 예술경영지원센터, 전국 문화재단 등)의 공고를 취합하는
애그리게이터라 순수예술 공공 지원사업은 사실상 커버되지만, 다음 영역은 빠져 있다.

- **콘텐츠·대중음악 계열**: 한국콘텐츠진흥원(KOCCA) — 대중음악·공연 제작지원 등
- **민간 재단·기업 후원**: 한국메세나협회, CJ문화재단, 두산연강재단 등
- 정부 직접 공고: 문체부 등 (→ 조사 결과 제외, 아래 참조)

## 결정 사항

| 항목 | 결정 |
| :--- | :--- |
| 아키텍처 | 소스 어댑터 구조 (`sources/` 디렉터리) |
| 1차 범위 | 어댑터 구조 도입 + `artnuri`(래핑) + `kocca`(신규) |
| 2차 범위 | 민간재단 (메세나협회 → CJ·두산 순, URL 조사 후) |
| 장르 범위 | 신규 소스는 문화예술 관련 공고만 (제목 키워드 필터) |
| 문체부 | **제외**. 목록에 접수기간 없음(게시일뿐), 포상·공시송달 등 잡음 다수, 마감일이 첨부 HWP에만 있는 경우 많음. 실제 지원사업은 산하기관 경유라 아트누리와 중복 |
| e나라도움 | **제외**. JS 셸이라 정적 수집 불가 |

## 아키텍처

```
sources/
  artnuri.js   ← 기존 crawler.js를 얇게 래핑 (전 장르, 기존 동작 그대로)
  kocca.js     ← 신규
auto.js        ← SOURCES 배열 순회 (등록·중복확인 흐름은 기존 유지)
crawler.js     ← 무변경 (index.js, cleanup.js 등 기존 스크립트 호환 유지)
```

### 어댑터 계약

모든 어댑터는 `fetchAll()` 하나를 노출한다. 필터링·제목 구성·설명 작성까지
어댑터 안에서 끝내고, 공통 스키마 배열을 반환한다. auto.js는 소스별 사정을 모른다.

```js
// fetchAll() → Promise<Item[]>
{
  sourceKey: 'ARTNURI' | 'KOCCA',   // 소스 식별자
  docId: string,                     // 소스 내 고유 ID (KOCCA는 intcNo)
  title: string,                     // 캘린더 제목 (프리픽스 포함 완성본)
  startDate: 'YYYY-MM-DD',
  endDate: 'YYYY-MM-DD',
  description: string,               // ID 라인 포함 완성본
}
```

- 각 어댑터는 단독 실행 가능해야 한다: `node sources/kocca.js` → 수집 결과 표 출력.
  필터 품질을 등록 전에 눈으로 확인하는 용도.

## 중복 방지 (2단계)

### 1단계 — 소스 내 (ID 기반)

이벤트 설명(description)의 ID 라인으로 판별한다.

| 소스 | ID 라인 형식 | 이유 |
| :--- | :--- | :--- |
| 아트누리 | `ID: CRL3569` (기존 유지) | 이미 캘린더에 등록된 기존 이벤트와 호환 |
| KOCCA 등 신규 | `ID: KOCCA:326D00091012` | 소스 간 ID 충돌 방지 (네임스페이스) |

### 2단계 — 소스 간 (제목 유사도 안전망)

아트누리 취합 목록에 KOCCA·정부·주요 민간재단이 없음을 확인했으므로 소스 간
중복은 드물다. 다만 안전망으로:

- 등록 직전, 시작일 ±1일 범위의 기존 이벤트 제목과 비교
- 정규화: `[프리픽스]` 제거 → 공백·기호 제거 → 2글자 단위 토큰화
- 토큰 Jaccard 유사도 ≥ 0.75이거나 한쪽 정규화 문자열이 다른 쪽에 포함되면
  등록하지 않고 로그에 `⏩ 유사 공고 존재` 남김 (임계값은 상수로 조정 가능)
- 기존 `findEvent`의 events.list 호출을 확장해 재활용 — **API 호출 수는 현행과 동일**

## KOCCA 어댑터 상세

- 목록: `kocca.kr/kocca/pims/list.do?menuNo=204104` — 정적 HTML, table 구조 (확인 완료)
- 카테고리: 자유공모(1)·지정공모(2)·모집공고(3) 수집, 종료된사업(4) 제외
- 접수기간: 목록에서 바로 파싱 (`26.07.13 ~ 26.07.31` → ISO). 상세 페이지 요청 불필요
- 고유 ID: 링크의 `intcNo` 파라미터
- 장르 필터: 목록·상세 모두 장르 메타데이터가 없으므로 **제목 키워드 include 필터**
  - 키워드(파일 상단 상수, 수정 가능): 음악, 대중음악, 공연, 예술, 전시, 미술, 문학,
    무용, 연극, 뮤지컬, 국악, 클래식, 인디, 밴드, 콘서트, 페스티벌, 뮤지션, 버스킹
  - include 방식인 이유: 게임·방송·웹툰 등 무관 공고가 다수라 exclude 목록으로는
    누수가 생긴다. 놓친 공고는 로그로 확인해 키워드를 보강한다
- 캘린더 제목: `[KOCCA] 공고명`
- '대관' 제외 필터는 아트누리와 동일하게 적용

## 에러 처리

- 소스별 try/catch: 한 소스가 실패해도 나머지는 진행
- 최종 요약에 소스별 통계: `ARTNURI: 신규 3 / KOCCA: 신규 1, 실패 0`
- 30분 워치독, 비대화형 인증(`interactive: false`) 등 기존 안전장치 유지
- launchd 설정 무변경 (진입점 auto.js 동일)

## 검증 계획

1. `node sources/kocca.js` 단독 실행 → 필터 통과/제외 목록 눈으로 확인
2. `node sources/artnuri.js` 단독 실행 → 기존 crawler와 동일 결과 확인
3. `node auto.js` 실제 실행 → 소스별 등록·중복 통계 확인
4. 재실행 → 신규 0건 (중복 방지 회귀 확인)
